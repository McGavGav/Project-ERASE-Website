## What's New
* Translation feature
* Account deletion
* Reports dashboard
* Event creation and RSVP system
* RSVP management in the admin panel
* Student database image uploads, live filtering, bulk uploads, and image cropping
* Workshop/Event map management and filtering
* Updated contact information page

## Work Summary
In this sprint the team focused on expanding the site's core features and connecting more of the front-end experience to working backend data. We added the first version of the translation feature, including Spanish locale files and updates across major templates. We improved user account functionality by adding account deletion and refining email/report-related behavior.

The event calendar was expanded from a visual calendar into a more functional event system. Admin users can create events, events render on the calendar, event descriptions appear in pop-ups, and users can RSVP to events that require reservations. The team also added backend RSVP tracking and admin-facing RSVP listing/detail pages so board members can review event participation.

The student database received additional media and usability improvements. Student records now support photo uploads, uploaded photos are removed when students are deleted, and the interface includes live filtering, bulk upload support, and image cropping. The map feature was also developed further with workshop map management, workshop data models, admin registration, and filtering support.

## Retrospective Summary
Here's what went well:
* The team completed several major client-facing features during the sprint
* Event calendar work progressed into a functional RSVP-backed system
* Student database usability improved with filtering, photo support, and upload tools
* Translation work created a foundation for bilingual site content
* Map work progressed into a manageable workshop/event system

Here's what we'd like to improve:
* Leave more time for integration testing after merging larger branches
* Keep files organized as models change across branches
* Continue validating workflows from both regular user and admin perspectives

Here are changes we plan to implement in the next sprint:
* Refactor and polish existing features
* Begin implementing links between features such as the calendar, reports dashboard, and interactive map
